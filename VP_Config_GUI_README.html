Visual Phaser V2.0 Configuration GUI

A wxPython graphical editor for VP_configV2.py — the configuration file for Visual_Phaser.V2.0.py. It lets you browse VCF files, select samples from VCF headers, edit all settings through purpose-built controls, validate inputs, and launch a run directly from the GUI.

Quick start: open a terminal in the visual-phaser folder and run python VP_Config_GUI.py. The GUI loads VP_configV2.py from the same folder automatically, or VP_configV1.py if V2 is not available.

Features

Path Browsers
Browse buttons for FILES_PATH, WORKING_DIRECTORY, and MAP_PATH.
File Lists
SIBLINGS, PHASED_FILES, EVIL_TWINS, COUSINS, and CHROMOSOMES as comma-separated fields.
Boolean Dropdowns
True/False combo boxes for every boolean parameter.
Spin Controls
Integer spin boxes for RESOLUTION, cutoffs, SNP minimums, and more.
Tooltips
Hover any field for a description drawn from the original config comments.
Unsaved-change Guard
The Run button is blocked until the configuration has been saved.
Live Output
Program stdout/stderr is streamed into the Program Output panel in real time.
VCF Validation
Sample names in SIBLINGS and COUSINS are validated against VCF header before running; mismatches are reported with available sample names.

Installation

Prerequisites

  • Python 3.8 or higher
  • wxPython 4.0 or higher
  • pandas, numpy, openpyxl, Pillow (required by the processor)

Install wxPython

pip install wxPython

On Linux install the system package instead if pip fails:

# Ubuntu / Debian
sudo apt-get install python3-wxgtk4.0

# Fedora
sudo dnf install python3-wxpython4

Install processor dependencies

pip install pandas numpy openpyxl Pillow

Usage

1 — Load a Configuration

  • At startup the GUI loads VP_configV2.py from its own directory automatically (falls back to VP_configV1.py if not found).
  • To load a different file: File → Load Config or click Load Configuration.

2 — Edit Settings

  • Switch between the four tabs: Paths, Input files, Generation options, Algorithm factors.
  • Use Browse buttons for folder paths.
  • Enter list fields as comma-separated values (for example: Alice, Bob, Carol).
  • Leave CHROMOSOMES blank to process all 23 chromosomes.
  • The status bar shows "Unsaved configuration changes" whenever a field is modified.

3 — Save & Run

  1. Click Save Configuration (or File → Save Config) to write changes to VP_configV2.py.
  2. VCF sample names in SIBLINGS and COUSINS are validated against the VCF file before starting.
  3. Click the green Run button to launch the processor. If there are unsaved changes or validation fails, a warning is shown.
  4. Watch progress in the Program Output panel on the Paths tab. Use Clear to reset it.

Supported Input File Formats

VCF files (Variant Call Format) must be placed in FILES_PATH as a single file or within a folder. The loader automatically detects gzip compression, parses the VCF header to extract sample column names, and validates that SIBLINGS and COUSINS names match available VCF samples.

SourceFormatNotes
VCF.vcf or .vcf.gz (gzip-compressed)Standard Variant Call Format; GT field decoded; REF/ALT used as fallback
VCF (multi-sample).vcf with multiple sample columnsSample names extracted from #CHROM header; each sample can be listed in SIBLINGS/COUSINS
Legacy formatsAncestryDNA, 23andMe, MyHeritage (raw DNA)V2.0 focuses on VCF; legacy format support available in V1.1
Y chromosome and mitochondrial (MT) rows are silently dropped. Only autosomes (1–22) and the X chromosome (mapped to 23) are analysed.

Troubleshooting

Run button shows "Save Required" warning

You edited one or more fields without saving. Click Save Configuration first.

"Input Data Error" popup when running

One or more SIBLINGS files could not be parsed into usable data. The popup lists each failing individual and the specific reason (missing file, unrecognised columns, no autosomal rows, etc.).

VCF file or samples not found

Check that FILES_PATH points to a valid VCF file or folder containing .vcf/.vcf.gz files. Verify sample names in SIBLINGS/COUSINS match the VCF header exactly (case-sensitive).

Empty/missing workbook after run

If no chromosome sheets can be created (e.g. all individuals fail to load), a Results sheet is written with a diagnostic message so the file can still be opened.

"IndexError: at least one sheet must be visible"

Fixed in current release via the ensure_visible_worksheet guard. Rebuild your exe if running a frozen version.

Documentation opens in Notepad

No default application is associated with .html files. Right-click the file and choose Open with → your browser, then set it as the default.

wxPython installation fails on Windows

pip install --upgrade wxPython

wxPython on macOS (Apple Silicon)

pip install wxPython --no-binary wxPython

Changes not saving

  • Confirm write permission to VP_configV2.py.
  • Make sure no other editor has the file locked.
