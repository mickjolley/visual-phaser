VISUAL PHASER CONFIG GUI - QUICK START GUIDE
================================================================================

INSTALLATION (First Time)
------------------------

1. Open Command Prompt / Terminal
2. Navigate to the visual-phaser directory, such as:
   cd C:\Users\r<user>>\Documents\GitHub\visual-phaser

3. Install wxPython:
   pip install wxPython>=4.0.0


LAUNCHING THE GUI
-----------------

Method 1: Simple Launch
   python VP_Config_GUI.py

Method 2: With Config File
   python VP_Config_GUI.py VP_configV1.py


USING THE GUI
-------------

1. LOAD CONFIGURATION
   - File menu → Load Config, OR
   - Click "Load Configuration" button
   - Select your VP_configV1.py file

2. EDIT SETTINGS
   Paths:        Click "Browse..." buttons to select folders
   Files:        Use "Select Files" or type names as comma-separated, one-per-line, or mixed entries
   Boolean:      Choose True/False from dropdown
   Numbers:      Use up/down spinners or type values
   Text:         Type directly in text fields

3. GET HELP
   - Hover over any field name to see a helpful tooltip
   - Tooltips contain the original configuration documentation

4. SAVE YOUR WORK
   - Click "Save Configuration" button, OR
   - File menu → Save Config
   - Your changes are written to VP_configV1.py

5. RESET / START OVER
   - Click "Reset to Defaults" button
   - Reloads the values from the currently open config file

6. Run
   - Click "Save Configuration" button if modifications have been made
   - Click the "Run" button to run the main program

FIELDS ORGANIZED BY CATEGORY
-----------------------------

PATHS AND DIRECTORIES:
  • FILES_PATH          - Where your DNA files are stored
  • WORKING_DIRECTORY   - Where output files will be saved
  • MAP_PATH            - Where min_map.txt is located

FILE MANAGEMENT:
  • SIBLINGS            - Comma-separated names of the individuals to compare (2 minimum)
  • PHASED_FILES        - Comma-separated names of the individuals in phased files
  • EVIL_TWINS          - Comma-separated names of the individuals in evil-twin files
  • COUSINS             - Comma-separated names of the individuals to compare with siblings
  • CHROMOSOMES         - Which chromosomes (1-23, or blank for all)
  • EXCEL_FILE_NAME     - Name for output file

DISPLAY OPTIONS:
  • CHROM_TRUE_SIZE              - Use true vs. normalized sizes
  • LINEAR_CHROMOSOME            - Show linearized view
  • SHOW_NO_MATCHES              - Include unmatched pairs
  • FIR_TABLES                   - Display FIR analysis tables

PROCESSING:
  • MERGE_FILES                  - Combine DNA data
  • AUTO_REC_PNTS                - Auto-calculate recombination points
  • AUTO_RP_ASSIGN               - Auto-assign recombination points
  • REPAIR_FILES                 - Fix isolated SNPs

QUALITY PARAMETERS:
  • RESOLUTION          - Display resolution (1-100)
  • ARP_TOLERANCE       - Minimum column width for recombination
  • HIR_CUTOFF          - Min HIR segment length (cM)
  • FIR_CUTOFF          - Min FIR segment length (cM)
  • X_HIR_CUTOFF        - X chromosome HIR cutoff (cM)
  • X_FIR_CUTOFF        - X chromosome FIR cutoff (cM)
  • HIR_SNP_MIN         - Minimum HIR SNPs
  • FIR_SNP_MIN         - Minimum FIR SNPs
  • MM_DIST             - Mismatch distance (Kbs)

ADVANCED:
  • SCALE_FACTOR        - Column width per pixel adjustment
  • SCALE_ON            - Enable/disable scale display
  • FREEZE_COLUMN       - Column to freeze in display
  • LINUX_FONT_STRING   - Font path (Linux only)
  • NO_CALL             - Character for missing data


KEYBOARD SHORTCUTS
------------------

Ctrl+O     Load Configuration
Ctrl+S     Save Configuration
Alt+F4     Exit Application


TROUBLESHOOTING
---------------

Q: "wxPython is not installed" error
A: Run: pip install -r requirements-gui.txt

Q: Changes don't seem to save
A: Make sure you clicked "Save Configuration" button or used File → Save Config
   Check that you have write permission to VP_configV1.py

Q: GUI appears frozen
A: Wait a moment - file dialogs can take time to open
   If it stays frozen, close and reopen (Ctrl+C in terminal, then run again)

Q: Can't find VP_configV1.py
A: Make sure you're in the correct directory when launching
   You can also use File → Load Config to browse for it
